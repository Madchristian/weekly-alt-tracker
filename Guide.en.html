Manual installation

Ready in three steps.

You do not need an addon manager. All that matters is that the folder is nested exactly once and that the TOC file sits directly inside it.

Download the ZIP

Download WeeklyAltTracker-0.4.2.zip from the official project page or directly from the author.

Extract the archive

Open the ZIP. Inside is the folder WeeklyAltTracker. Copy exactly that folder.

Paste it into WoW

Paste the folder into _retail_\Interface\AddOns and restart WoW. If WoW is already running, use /reload afterwards.

First run

Log in. Record. Compare.

WoW allows no live access to logged-out characters. Every character therefore builds its own safe offline snapshot when it logs in.

Recommended order

  1. Enable the addonOn the character selection screen, check under “AddOns” that WeeklyAltTracker is enabled.
  2. Open the windowClick the minimap button or type /wat in chat.
  3. Refresh the snapshotClick Refresh now in the Settings section once the Great Vault, quest log and profession data have loaded.
  4. Log in each alt onceRepeat this for every character that should appear in the account-wide overview.
  5. Enter a delveOnly there can Blizzard provide the Gilded Stash progress for the first time.

Language

The interface follows your WoW client language automatically: deDE is fully German, enUS and enGB are fully English.

Every other client language safely uses the English version. There is deliberately no separate setting.

Names that come from the game – class, dungeon, item, profession and achievement – always come from WoW itself and are never translated by the addon.

Minimap button

Left click opens or closes WeeklyAltTracker.

Dragging with the left mouse button moves the button along the minimap edge. The position is saved automatically.

The button uses no external library and works independently of other addons.

Feature set

Seven views. One weekly picture.

The interface stays deliberately compact. Each view answers one concrete question instead of squeezing all data into one overly wide table.

06

Statistics

Since 0.4.0 shows thirteen lifetime values – delves, dungeons, playtime, deaths, healthstones and quests. Since 0.4.2 the page is no longer a side-by-side comparison of all characters but a per-scope dashboard: a fixed register bar along the bottom selects the scope – pinned at the far left is TOTAL (the account total), and to its right one tab per known character. From the eighth character onwards the character tabs live in a horizontally paged viewport with explicit arrow buttons; TOTAL always stays pinned and never pages away. The active character tab carries its class colour, the active TOTAL tab the turquoise of the interface. Above the bar all thirteen values of the selected scope are shown simultaneously as metric cards in three sections: content (delves, Midnight delves, dungeons entered, Midnight dungeons, playtime), survival (deaths total, in dungeons, in raids, from falling, healthstones) and quests (completed, daily, abandoned). Every card visibly binds a concise label to a prominent value and clips hard – no value can bleed into its neighbour, at any scale preset. Very large lifetime values are abbreviated on the card (for example 123T); the tooltip still states the exact full value, small values stay exact and playtime is never abbreviated. Dungeons entered counts 5-player dungeons entered, not completed; the tooltip says so explicitly. Midnight dungeons is not a single Blizzard statistic but the sum of the 24 final boss statistics of the eight Midnight dungeons across Normal, Heroic and Mythic – it only appears when all 24 components are safely readable. Only safely known values are summed; if no value is known, “−” is shown and never an invented zero.

Understanding status

Colours with a clear meaning.

WeeklyAltTracker deliberately distinguishes between open, partially done and unknown. Missing data is never invented as a real zero.

Green

Done, unlocked or safely completed.

Yellow

Partially done or still in progress.

Red

Safely open or not yet completed.

Grey / −

Unknown, not recorded or from an old week.

Chat commands

Direct access with /wat.

Since version 0.3.0 there is only one public command. Every option lives in the Settings section of the left navigation. The command itself is identical in German and English; only its output is translated.

/watOpen or close the window.
/weeklyaltEquivalent alias.
/wat <anything>Opens the window directly in the Settings section. The former subcommands show, hide, refresh, resetpos and scale moved there without replacement.

Help & limits

When something does not show up yet.

Most apparent errors come from a doubly nested folder, a character that has not logged in yet, or data that Blizzard only provides after a certain interface has been opened.

The addon does not appear on the character selection screen

Check that WeeklyAltTracker.toc sits directly in the folder AddOns\WeeklyAltTracker. An extra folder such as WeeklyAltTracker\WeeklyAltTracker is wrong. Also check that you really extracted the archive.

A character or new fields are missing

Log in the character in question once with the addon enabled and click Refresh now in the Settings section. Logged-out characters cannot be queried live by WoW.

The Gilded Stash shows “not tracked”

Enter a delve at least once with this character. Blizzard provides the corresponding UI widget only there or nearby. A value missing later does not overwrite an already safe snapshot.

Vault item level or other values show “−”

“−” means unknown, not loaded or protected – not zero. Open the Great Vault, wait briefly for the item data and then click Refresh now in the Settings section. Statistics likewise only appear once WoW has loaded the achievement data.

Which bags are scanned for profession knowledge?

Backpack, four normal bags and the reagent bag. Bank and warband bank are deliberately not scanned. Stackable knowledge items are counted with their quantity.

Can I switch the language?

Not in the addon itself – it follows your WoW client language. If you switch the client to German, the interface appears fully German after restarting the WoW client, including characters already recorded. That works because the addon no longer stores its own translation labels as the display source: it stores stable IDs and localizes them only when displaying, and that runtime resolution takes precedence. Already-localized names supplied by WoW may still sit in the snapshot, but only for backwards compatibility and as a fallback; if no localization is available you see a neutral Dungeon ID instead of a name stored in another language.

Why is there no raid tracking?

The addon is deliberately tailored to Mythic+, Delves/World, Midnight weekly activities and alt management. Raid progress and the raid vault stay fully excluded.

How do I update the addon manually?

Close WoW as completely as possible, replace the folder AddOns\WeeklyAltTracker with the new version and start the game again. Your data lives separately in WoW’s SavedVariables and is preserved during a normal update.

A Lua error occurs

Note the exact error or copy it from BugSack/!BugGrabber. Report the version, the affected view and the last step you performed via the official project page.

Privacy

Your data stays in your WoW folder.

WeeklyAltTracker works entirely locally. The addon has no telemetry, no advertising, no network communication and transmits no character data to the author or any other service.
